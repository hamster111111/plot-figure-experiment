权威 Plotting Bench 与 Framework Landscape

面向当前方向筛选 plotting / chart generation 相关的 benchmark 与 framework，重点区分“真 plotting”、“方向相反的 chart-to-code”、以及“看起来相关但其实是 illustration”。 Last updated: 2026-04-19

当前最该盯的主 bench

Text2Vis 和 VisEval 是现在最适合当前思路的两块主实验地。前者最贴 answer + chart 一致性，后者最像社区权威 NL2Vis benchmark。

补充 bench

PandasPlotBench、PaperBananaBench/plot、MatPlotBench 都有用，但更适合补充实验、ablation、case study，而不是单独撑主叙事。

不建议混进主实验

FigureBench、SciGenBench、ChartMimic、Plot2Code 看起来相关，但要么偏 illustration，要么方向是 chart image -> code，不适合当主 plotting bench。

当前方向的推荐实验栈

如果继续 Plan2Chart

主实验：Text2Vis + VisEval

补充：PandasPlotBench + PaperBananaBench/plot

小规模 case study：MatPlotBench

原因：Plan2Chart 的核心是 answer + chart + shared analytical plan。只有 Text2Vis 真正在奖励这个贡献，VisEval 则提供权威 NL2Vis 外部验证。

如果还保留 F-A

最适合的角色：补充模块，不再当主方向。

可测场景：PaperBananaPlot 自定义 AST / provenance 指标，MatPlotBench 小样本 case study。

风险：很多 bench 最终只看图像，不直接奖励 provenance / traceability / transform correctness。

不建议作为主实验

FigureBench：偏 illustration，不是 plotting。

SciGenBench：scientific image 太混，plot 信号被稀释。

ChartMimic / Plot2Code：方向是 chart image -> code，和 data-to-chart 不同。

批判性提醒
  • 不要把 chart-to-code 和 data/query-to-chart 混成一类。它们都叫 plotting，但研究问题不同。
  • 不要把 scientific illustration 和 data plotting 混成一类。FigureBench、AutoFigure 这一支和当前问题只有表面相似。
  • 不要只看 benchmark 名字里的 “figure / chart / plot”。一定要看输入是什么、输出是什么、原生指标奖励什么。
  • 如果方法核心卖点是 shared analytical plan 或 reasoning-to-render consistency，那就必须优先选能测答案和图表联动的 benchmark，而不是只看最终图像相似度。
