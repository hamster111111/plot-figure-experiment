生成的图表数据点数量多于 Ground Truth。通常是因为模型没有做 GROUP BY 聚合，直接画了原始数据的每一行。

vis_id: 1003
PieMedium
NL Query
Show the product name and total order quantity for each product with a pie chart.
GT SQL
SELECT T1.product_name, SUM(T2.order_quantity) FROM products AS T1 JOIN order_items AS T2 ON T1.product_id = T2.product_id GROUP BY T1.product_name
生成代码
ax.pie(df['product_price'], labels=df['product_name'], autopct='%1.1f%%')
GT Data
x=["Apple","gucci","jcrew"], y=[23,21,40] (3 groups)
Rationale
visualization data length 15 != ground truth length 3.
Code Issue
wrong_agg_count_vs_sum
生成的图表（15 个数据点，应为 3 个）
模型用了 product_price 列而非 SUM(order_quantity)，且没有 GROUP BY，导致 15 个数据点 vs GT 的 3 个分组。
vis_id: 102@y_name@ASC
BarHard
NL Query
Return a bar chart about the number bookings for each apartment number, I want to list y axis in ascending order.
GT SQL
SELECT apt_number, COUNT(apt_number) FROM Apartment_Bookings AS T1 JOIN Apartments AS T2 ON T1.apt_id = T2.apt_id GROUP BY apt_number ORDER BY COUNT(apt_number) ASC
生成代码
df['apt_number'].value_counts()
GT Data
10 apartments with counts [1,1,1,1,1,1,1,2,3,3]
Rationale
visualization data length 15 != ground truth length 10.
Code Issue
missing_groupby
生成的图表（15 行原始数据 vs GT 10 个分组）
模型没有做 JOIN + GROUP BY，直接对单表做 value_counts，得到 15 行而非 10 个分组。
