MatPlotBench — 不适合作为核心证据

✗ MatPlotBench 评的是视觉相似度，不是数据正确性

  • 结论：不能用 MatPlotBench 证明"现有系统有数据保真度问题"
  • 可以用来证明"F-A 不损害视觉质量"（修复后分数不下降），这是可行的次级主张

推荐论文叙事结构（Abstract A 版）

1

Problem / Scene（1-2 句）

LLM 能生成可执行、视觉合理的图表代码，但可能 silently 违反 aggregation/filter/join/sort 约束。这类错误在现有评测指标下不可见。

2

Limitation（1-2 句）

现有系统优化 executability 或 visual similarity，没有显式中间表示，缺少执行后的数据语义验证。VisEval 在 strict data_check 口径下 36.8% pass rate vs 宽松口径 85.2%——48.4pp 落差全来自数据保真度。

3

Method（2-3 句）

我们提出 GroundedChart：从 NL query 抽取操作符级意图（Intent Parser），用 CanonicalExecutor 计算期望绘图数据，再用 OperatorLevelVerifier 与实际执行结果做逐算子对比，最后通过 localized repair 修复错误并重新验证。

4

Results（2-3 句）

在 VisEval 上，GroundedChart 将 Full Pass Rate 从 36.8% 提升至 XX%（还需跑实验），同时在去掉 data_check 的口径下维持 85%+ 的视觉质量。在 PlotCraft 上泛化至 48 种图表类型。

→ 论文需要展示 operator 级别的细粒度分析，单一 groupby 修复不够
⑤

Smoke Test 跑出来 confidence=0.35，说明 HeuristicParser 的规则过于保守。需要分析 VisEval 的 NL query 语料，改进关键词匹配规则和 schema-aware 推断，让 Heuristic baseline 也足够强，才能衬托 LLM Parser 的提升。

→ 消融实验的 baseline 要足够强，否则"LLM Parser 提升了"这个 claim 也站不住

→ 只有 VisEval 一个 benchmark，论文泛化性会被质疑
⑦

把 EvidenceGraph 做成可视化 + 论文 Figure

EvidenceGraph（"哪段原文 → 哪个验证错误"的关联图）是框架最有特色的部分，但目前只有数据结构，没有可视化。做一个案例可视化图放进论文，能极大提高论文的可读性和说服力。

长期方向建议（P2）
