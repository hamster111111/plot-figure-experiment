Abstract A · Execution-Grounded Verification-and-Repair
适合你现在这条主线：以 faithful language-to-chart generation 为题，核心证据主要靠 VisEval，Text2Vis/MatPlotBench 只做补充。

Large language models can generate chart code from natural-language requests and structured data, but existing systems are typically optimized for executability or visual quality rather than whether the rendered figure faithfully reflects the requested data operations. As a result, models can produce charts that look plausible while encoding incorrect filters, group-bys, aggregations, or sort orders. We study this problem as faithful language-to-chart generation and present GroundedChart, an execution-grounded verification-and-repair framework. GroundedChart first extracts a structured chart requirement plan from the user request, including data-operation and basic encoding constraints. It then derives expected artifacts from the source data with deterministic execution and compares them against traces recovered from the generated plotting program, enabling requirement-level mismatch detection and localized repair. Unlike black-box LLM judging, our framework grounds verification in executable evidence and attributes each failure to specific requirements. We evaluate GroundedChart on benchmarks for natural-language-to-chart generation, using VisEval as the primary testbed and supplementary settings from Text2Vis and MatPlotBench. Results show that GroundedChart improves chart fidelity and data-check success while preserving executability, and provides interpretable error reports for downstream analysis. Our findings suggest that semantic alignment between language intent, data transformation, and plotted traces is a distinct bottleneck in chart generation that cannot be resolved by stronger base models alone.

为什么这版稳 claim 收得比较准，重点放在 data operation fidelity、execution-grounded verification、localized repair，和你当前已经形成的证据链最贴。
主要风险 如果最后实验只修了少数 groupby/filter 错误，这版还能保住；但“basic encoding constraints”那句最好确保你系统至少有初步覆盖。
建议：如果明天就要拿给师兄看摘要，先看这版。
强 claim Framework-first 要求更高
Abstract B · Chart Requirement Fidelity
这版把故事从“数据保真”进一步扩展为 chart requirement fidelity，适合你后面真的把 encoding / order / presentation constraint 也做进来。

Natural-language chart generation involves more than producing runnable code: the final figure must satisfy chart requirements expressed or implied in the request, including data transformations, visual encodings, and presentation constraints such as ordering or value formatting. Existing systems lack an explicit representation of these requirements and therefore struggle to diagnose silent fidelity errors, where a chart executes successfully but violates the intended semantics. We introduce GroundedChart, a requirement-grounded framework for faithful language-to-chart generation. The framework converts a user request into a traceable requirement plan, computes expected artifacts with deterministic programs, extracts actual plot traces from generated code, and performs requirement-level verification before localized repair. This design turns chart correctness from an opaque end result into a structured, evidence-backed process linking query spans, requirement nodes, computed artifacts, plotted traces, and repair actions. We further define chart requirement fidelity as an evaluation perspective that complements executability and visual similarity. Across benchmark settings, GroundedChart produces more faithful chart code, reveals failure modes that are invisible to surface-level metrics, and supports fine-grained error analysis by requirement family. These findings suggest that structured requirement grounding is an effective foundation for building trustworthy chart generation systems.

为什么这版强 这版最像“提出了一个新任务视角 + 新框架 + 新分析维度”的顶会摘要，理论高度更高，不只是 repair pipeline。
主要风险 reviewer 很可能会问：你到底支持了多少 requirement family？如果现在主要还是 data ops，这版会显得略微超卖。
建议：只有当 requirement extraction、constraint verification、family-wise error analysis 都做得比较完整时再上这版。

Abstract C · Evidence-Grounded and Traceable
这版最适合你想把“可溯源、可归因、LLM 不负责算数真值”打成系统特色的时候，用来区别普通 agent framework。

Many chart-generation errors are silent: a plotting program executes and produces a visually plausible figure, yet the underlying data no longer corresponds to the user’s request. Existing systems and benchmarks provide limited support for tracing such failures back to their causes. We present GroundedChart, an evidence-grounded framework that makes chart generation verifiable and attributable. Given a natural-language request, source data, and candidate chart code, GroundedChart extracts requirement nodes with provenance, computes expected artifacts via deterministic execution, aligns them with actual plot traces, and constructs an evidence chain for requirement-level verification and repair. This approach separates numerical truth from language modeling: LLMs are used for requirement interpretation and code revision, while filters, joins, aggregations, percentages, and ordering are checked by executable programs. The resulting verifier can abstain under ambiguity, localize mismatches to specific requirements, and prevent unconstrained rewrites during repair. Experiments on language-to-chart benchmarks show that evidence-grounded verification improves fidelity, produces more actionable diagnostics, and enables analysis of which requirement families remain challenging. Our study argues that traceability, not just generation quality, is central to building reliable chart-generation systems.

为什么这版有特色 它把你的系统从“又一个 chart agent”拉成“evidence-backed, traceable verification framework”，很利于和黑箱 judge / 黑箱 self-refine 区分。
主要风险 这版默认你的 evidence chain、abstention、localized repair 都至少有 prototype。若实现不够硬，摘要会比系统本体更强。
建议：如果你的论文想主打 methodology 和分析可信度，这版比 A 更有“味道”。
当前推荐

如果按你现在的状态，我建议：

  • 主摘要先用 A：最贴近当前证据结构，不容易被 reviewer 一眼看出 overclaim。
  • 如果接下来 traceability 真做扎实，再切到 C：这会让论文辨识度更强，也更像你自己的方法，而不是通用 repair agent。
  • B 先保留：它代表你最理想的论文终局，但只有 requirement family 做得够广时才建议上。

更直白一点：A 是现在能 defend 的版本，C 是最值得冲的版本，B 是需要更多实现才能站稳的版本。
