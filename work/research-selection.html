Benchmark 选型建议

基于三个 benchmark 的复现数据、SOTA 对比、以及对 F-A 方向的支撑力，给出选型建议。

Plotting Benchmark 全景（17 个候选）

从 32 篇相关论文中筛选出 17 个 benchmark，逐一核实其实际内容后按 F-A 相关度重新排序。详细卡片见 Benchmark 卡片页。

重要修正：经逐一查阅论文后发现，SciGenBench、FigureBench、PaperBananaBench、SciFigBench 实际上是科学插图（架构图/流程图/分子结构等）benchmark，不是数据驱动图表 benchmark。之前基于名称和摘要的分类有误，现已修正。ChartMimic 虽然声称有"数据保真度"指标，但实际评测中并无逐点数据对比。PlotCraft 与 VisEval 是独立数据集，非同源。

Plot/Figure — 数据驱动图表（7 个）

VisEval, PlotCraft, MatPlotBench, Text2Vis, PandasPlotBench, nvBench, nvBench 2.0

输入是 NL + 结构化数据（CSV/数据库/DataFrame），输出是绑定真实数据的图表代码。数据保真度问题在这类任务中最突出。

F-A 的主战场。其中只有 VisEval 内置了 data_check。

Image → Code（5 个）

ChartMimic, Chart2Code, RealChart2Code, PlotGen-Bench, Plot2Code

输入是图表图片，输出是复现代码。核心挑战是视觉理解。ChartMimic 声称有数据保真度指标，但实际只有视觉相似度 + 文本/布局/类型/颜色 F1，无逐点数据对比。

Illustration — 科学插图（4 个）⚠️ 之前误分类

SciGenBench, FigureBench, PaperBananaBench, SciFigBench

输入是论文文本/科学问题描述（无结构化数据），输出是架构图、流程图、几何图、电路图、分子结构等。与 F-A 完全正交——"数据保真度"概念不适用。

之前基于名称误判为数据图表 benchmark，现已修正。

Chart Editing（1 个）

ChartEditBench

输入是现有图表 + NL 编辑指令，输出是修改后的代码。与 F-A 的生成场景不同。

VisEval — 推荐作为主实验 Benchmark
我们的复现数据
  • 1,150 实例，Pass Rate 36.8%（含 data_check）
  • 953 例 data_check 失败，其中 72% 是数据保真度问题
  • 口径转换：去掉 data_check 后 85.2%，与 Gemini-2.5-Pro (83.71%) 持平
  • 48.4pp 的落差全部来自 data_check——精确量化了数据保真度瓶颈
SOTA 对比
  • SOTA: Claude-4.1-Opus 96.14%（PlotCraft, 2025）— 但不含严格 data_check
  • 开源 SOTA: PlotCraftor-30B 93.81%（基于 Qwen3-Coder）
  • GPT-4 baseline: 75.27%（VisEval 原文, 2024）
  • 同口径下我们的 85.2% 处于 GPT-4 ~ Gemini 水平
选型理由：VisEval 是唯一内置 data_check 的 benchmark，能直接量化 F-A 的改进效果。SVG 解构评测方式精确到每个数据点，不依赖 LLM 打分。口径转换数据（36.8% → 85.2%）是论文中最有力的论据之一。7 种图表类型覆盖主流场景。
注意事项
  • 仅 7 种图表类型（bar, line, pie, scatter, area, heatmap, box），不含 3D、地图等
  • PlotCraft 等论文的 pass rate 不含严格 data_check，直接对比数字会产生误导——论文中需要明确说明口径差异
  • 建议：在 GPT-4 / Claude 上也跑一遍，证明 data_check 失败不只是弱模型的问题
Text2Vis — 推荐作为补充实验
我们的复现数据
  • 1,892 样本，Final Pass 33.4%，Answer Match 38.1%
  • 1,309 例失败中 81% 是 answer_mismatch
  • answer_mismatch 中：wrong_entity 51.2% + wrong_value 33.1% = 84%
  • 但大部分是推理/提取错误，不是图表数据错误（很多案例 readability 5.0, correctness 5.0）
SOTA 对比
  • SOTA: GPT-4o Agentic 42%（Text2Vis 原文, EMNLP 2025）
  • 开源 SOTA: RL-Text2Vis-14B 29%（Qwen-2.5-14B + GRPO, EACL 2026）
  • 我们的 33.4% 超过 GPT-4o Direct (26%) 和 Gemini-2.0-Flash (26%)
  • 即使 SOTA 也只有 42%——这个 benchmark 对所有模型都很难
选型理由：20+ 图表类型、难度分级（medium/hard/extra hard）、端到端 QA 验证。规模大（1,892 样本），统计显著性好。我们的成绩超过 GPT-4o Direct，说明复现质量可靠。
诚实评估 — F-A 相关度有限
  • 81% 的失败是 answer_mismatch，但其中大部分是推理/提取错误，不是图表数据错误
  • F-A 能直接解决的可能只有 ~19% 的失败（exec_failure + low_correctness + 部分 wrong_value）
  • format_mismatch (92) + pct_vs_decimal (42) = 134 例是评测标准问题，不是真正的错误
  • 论文中不能把 33.4% 的 pass rate 直接拿来论证 F-A 的必要性
  • 建议用法：作为"端到端泛化验证"，展示 F-A 在更广泛任务上的效果，但不作为核心论据
MatPlotBench — 可作为泛化验证
我们的复现数据
  • 100 样本，Avg Score 56.4，Pass Rate 57%
  • 43% 低于 50 分，7 例得 0 分（完全错误主题/数据）
  • 失败以视觉还原度为主（颜色、布局、样式差异）
  • 只有 0 分案例可能涉及数据保真度
SOTA 对比
  • SOTA: VisPath (GPT-4o mini) 66.12（arXiv 2025）
  • PlotGen (GPT-4): 65.67（ACM IUI 2025）
  • MatPlotAgent (GPT-4): 61.16（ACL 2024）
  • 我们的 56.4 介于 Magicoder-6.7B (51.7) 和 MatPlotAgent-GPT-4 (61.16) 之间
  • CoDA (Gemini-2.5-Pro): ~80 VSR（ICLR 2026），但用不同指标
选型理由：竞品最多的 benchmark（MatPlotAgent, VisPath, PlotGen, CoDA 都在上面跑过），横向对比价值高。GPT-4o 打分虽然主观，但业界认可度高。
局限性
  • 样本量太小（100），统计显著性差，单个样本的分数波动影响大
  • 评测维度是视觉相似度，不是数据正确性——和 F-A 的目标基本不重叠
  • GPT-4o 打分有主观性，不同 run 可能有差异
  • 建议用法：仅作为"F-A 不损害视觉质量"的验证，不作为核心指标
推荐选型策略
角色Benchmark论文中的用途核心指标
主实验VisEval量化 F-A 对数据保真度的改进；口径转换对比 SOTAFull Pass Rate (含 data_check) + 口径转换 Pass Rate
主实验PlotCraft48 种图表类型广覆盖；对比 PlotCraftor SOTA；独立数据源验证泛化Pass Rate + 自有评测
补充实验Text2Vis端到端泛化验证；展示 F-A 在更广泛任务上的效果Final Pass Rate + Answer Match
泛化验证MatPlotBench证明 F-A 不损害视觉质量；横向对比竞品Plot Score
论文叙事建议
  • 核心故事线围绕 VisEval 展开：36.8% → 85.2% 的口径转换是最有力的论据
  • Text2Vis 作为补充：展示 F-A 在端到端任务上也有帮助，但不夸大
  • MatPlotBench 作为 sanity check：证明加了数据验证不会让图变丑
  • 主动说明 Text2Vis 大部分失败不是数据保真度问题——诚实比被 reviewer 质疑好
下一步实验建议
  • 优先级 1：在 GPT-4 / Claude 上跑 VisEval，证明 data_check 失败不只是弱模型的问题
  • 优先级 2：实现 F-A prototype，在 VisEval 上测 data_check pass rate 的提升
  • 优先级 3：复现 PlotCraft（独立数据源，48 种图表类型），验证 F-A 在不同数据集上的泛化
  • 优先级 4：在 Text2Vis 上测 F-A 对 exec_failure + low_correctness 的改善
  • 可选：PandasPlotBench (175 样本) 作为小规模快速验证
  • 注意：SciGenBench / FigureBench / PaperBananaBench / SciFigBench 是 illustration benchmark，与 F-A 正交，不建议投入
各 Benchmark 指标口径对照

不同 benchmark 和论文使用不同的评测指标，直接对比数字容易产生误导。以下梳理各指标的含义和可比性。

指标名称使用者含义含数据保真度？
Pass Rate (Full)我们的 VisEval 复现execution + surface-form + deconstruction + chart_type + data_check + order_check 全部通过是
Pass Rate (PlotCraft)PlotCraft 等论文validity (可执行) + legality (图表类型正确)，≈ 我们的 no_data_order 口径否
Final PassText2VisCode Exec 成功 + Answer Match 正确 + Correctness ≥ 3.5间接
Answer MatchText2Vis从图表提取的答案与 GT 答案匹配间接
Plot ScoreMatPlotBenchGPT-4o 视觉对比打分 (0-100)，含数据值、结构、颜色、标签等部分
VSRCoDAVisual Similarity Rate，图像级相似度否
ReadabilityVisEval / PlotCraft图表可读性评分 (1-5)否
核心发现：在所有主流评测指标中，只有我们的 VisEval Full Pass Rate 包含严格的数据保真度检查。PlotCraft 报告的 96.14% SOTA 和我们的 36.8% 不在同一口径上——同口径下我们是 85.2%。这个差异本身就是 F-A 最好的论据：现有评测体系系统性地忽略了数据保真度。
